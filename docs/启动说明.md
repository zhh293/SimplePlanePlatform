# 启动与运行说明

本项目是一个基于 Dubbo 微内核思想的加密隧道代理框架，运行时由两个独立进程组成：

- **proxy-remote**（远程服务端）：部署在能直连目标网站的机器上，接收加密隧道流量并代为访问目标站点。
- **proxy-local**（本地客户端）：部署在使用者本机，对外暴露 SOCKS5 / HTTP CONNECT 代理端口，把浏览器/系统流量经加密隧道转发给远程服务端。

数据流：`浏览器/系统 → proxy-local（本机 1080）→ 加密 HTTP/2 隧道 → proxy-remote（9090）→ 目标网站`。

因此正确的启动顺序是：**先启动 proxy-remote，再启动 proxy-local**。

---

## 1. 环境要求

| 项 | 要求 |
|----|------|
| JDK | 1.8 及以上 |
| Maven | 3.6 及以上 |
| 操作系统 | macOS / Linux / Windows 均可（系统代理自动设置分别走 networksetup / gsettings / 注册表） |

确认环境：

```bash
java -version
mvn -version
```

---

## 2. 构建

在项目根目录执行多模块构建：

```bash
mvn clean package -DskipTests
```

构建成功后产物位于各模块的 `target/` 目录：

- `proxy-local/target/proxy-local-1.0.0-SNAPSHOT.jar`
- `proxy-remote/target/proxy-remote-1.0.0-SNAPSHOT.jar`

> **注意（关于可运行 jar）**
> 当前两个模块只配置了 `maven-jar-plugin` 写入 `Main-Class`，**没有配置 shade / assembly 插件打 fat-jar，manifest 中也没有 Class-Path**。
> 所以直接 `java -jar proxy-local/target/xxx.jar` 会因为缺少 Netty / SnakeYAML / Logback 等依赖类而启动失败（`NoClassDefFoundError`）。
> 在补上 fat-jar 插件之前，请使用下面 **第 3 节** 的方式启动。

---

## 3. 启动（推荐方式）

下列命令均在**项目根目录**执行。

### 3.1 启动远程服务端 proxy-remote（先启）

```bash
mvn -pl proxy-remote -am exec:java -Dexec.mainClass=com.proxy.remote.ProxyRemoteServer
```

启动成功会看到类似日志：

```
=== ProxyRemoteServer started successfully ===
Listening on 0.0.0.0:9090, bizThreads=200, connectTimeout=5000ms
```

- 入口类：`com.proxy.remote.ProxyRemoteServer`
- 配置文件：`proxy-remote/src/main/resources/remote.yml`（从 **classpath** 加载，固定文件名 `remote.yml`，不接受命令行路径参数）
- 默认监听 `0.0.0.0:9090`

### 3.2 启动本地客户端 proxy-local（后启）

```bash
mvn -pl proxy-local -am exec:java -Dexec.mainClass=com.proxy.local.ProxyLocalServer
```

启动成功会看到类似日志：

```
=== Proxy Local Server started on port 1080 ===
  SOCKS5 proxy: localhost:1080
  HTTP  proxy:  localhost:1080
```

- 入口类：`com.proxy.local.ProxyLocalServer`
- 配置文件：`proxy-local/src/main/resources/proxy.yml`
- 默认监听本机 `1080` 端口，**同一端口同时支持 SOCKS5 与 HTTP CONNECT**（首字节嗅探自动识别）
- 若 `proxy.yml` 中 `systemProxy.enabled: true`，**启动时会自动把操作系统全局代理指向本机端口，进程退出（shutdown hook）时自动还原**

> proxy-local 的 `main` 方法支持通过**第一个命令行参数**指定外部配置文件的绝对路径；不传则从 classpath 加载 `proxy.yml`。
> 用 exec 方式传参示例：`mvn -pl proxy-local -am exec:java -Dexec.mainClass=com.proxy.local.ProxyLocalServer -Dexec.args="/abs/path/to/proxy.yml"`

## 4. 用瘦 jar + 手动 classpath 启动（备选）

如果不想用 Maven 插件、希望直接用 `java` 命令运行，可以让 Maven 把依赖拷到本地，再手动拼 classpath：

```bash
# 1. 把所有依赖拷贝到各模块 target/dependency 下
mvn -DskipTests dependency:copy-dependencies

# 2. 启动远程端（注意 classpath 同时包含本模块 jar、依赖目录、resources 目录）
java -cp "proxy-remote/target/classes:proxy-remote/target/dependency/*" \
  com.proxy.remote.ProxyRemoteServer

# 3. 启动本地端
java -cp "proxy-local/target/classes:proxy-local/target/dependency/*" \
  com.proxy.local.ProxyLocalServer
```

> Windows 下 classpath 分隔符为 `;`，路径分隔符为 `\`，请相应替换。

## 5. 关键配置对照

### 5.1 proxy-local（`proxy.yml`）

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `localPort` | 1080 | 本机代理监听端口 |
| `remoteServers[].host/port` | 127.0.0.1 / 9090 | 远程服务端地址，需指向 proxy-remote 的监听地址 |
| `remoteServers[].cipher` | none | 加密算法，**必须与服务端一致** |
| `remoteServers[].cipherKey` | my-secret-key-123 | 加密密钥，**必须与服务端一致** |
| `cluster` | failover | 集群容错策略 |
| `loadBalance` | roundrobin | 负载均衡策略 |
| `connectionsPerNode` | 2 | 每个远程节点建立的连接数 |
| `httpProxyEnabled` | true | 是否在同端口支持 HTTP CONNECT |
| `systemProxy.enabled` | true | 启动时是否自动设置系统全局代理 |

### 5.2 proxy-remote（`remote.yml`）

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `host` / `port` | 0.0.0.0 / 9090 | 监听地址与端口 |
| `bizThreads` | 200 | 业务线程池大小 |
| `cipher` | none | 加密算法，**必须与本地端一致** |
| `cipherKey` | my-secret-key-123 | 加密密钥，**必须与本地端一致** |
| `outbound.connectTimeoutMs` | 5000 | 连接目标站点超时 |
| `outbound.activeWaitTimeoutMs` | 5000 | DATA 到达时等待出站连接就绪的超时 |

> **两端必须对齐的两项**：`cipher` 和 `cipherKey`。任一不一致都会导致隧道解密失败、连接不可用。

---

## 6. 验证是否跑通

两端都启动后，用 curl 经本地 SOCKS5 端口访问外网：

```bash
curl -x socks5h://127.0.0.1:1080 https://www.example.com -v
```

或经 HTTP CONNECT：

```bash
curl -x http://127.0.0.1:1080 https://www.example.com -v
```

能正常返回页面即表示链路打通。也可以观察：

- proxy-local 日志：`logs/proxy-local.log`
- proxy-remote 日志：标准输出（按 `proxy-remote/src/main/resources/logback.xml` 配置）

---

## 7. 关闭

两端均注册了 JVM shutdown hook，直接 `Ctrl + C` 即可优雅关闭：

- **proxy-local** 关闭时会**先还原系统代理**（避免关闭后断网），再依次关闭监听通道、EventLoopGroup、超时时间轮与所有 ExchangeClient 连接。
- **proxy-remote** 关闭时会关闭所有出站会话、ExchangeServer 与线程池。

> 如果 proxy-local 异常退出（例如被 `kill -9`）来不及执行 shutdown hook，系统代理可能未被还原，需手动到系统网络设置中关闭代理。

## 8. 常见问题

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| `java -jar xxx.jar` 报 `NoClassDefFoundError` | jar 未打包依赖（见第 2 节注意） | 改用第 3 节 `exec:java` 或第 4 节手动 classpath |
| 本地端启动报连接失败 | proxy-remote 未启动或地址/端口不对 | 先启动远程端，核对 `remoteServers` 配置 |
| 能连接但数据乱码 / 隧道无响应 | 两端 `cipher`/`cipherKey` 不一致 | 对齐两个配置文件的加密算法与密钥 |
| 关闭后本机无法上网 | proxy-local 被强杀，系统代理未还原 | 手动到系统设置关闭代理 |
| 端口被占用 | 1080 / 9090 已被其他进程占用 | 修改 `localPort` / `port` 配置 |
