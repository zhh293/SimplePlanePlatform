# Product

## Register

product

## Users

Developers and ops engineers managing Netty-Proxy infrastructure. They need to configure proxy-local (client) and proxy-remote (server) instances, edit YAML parameters, monitor connection status, and manage route rules. Comfortable with technical tooling, expect precision and speed over decoration.

## Product Purpose

A web dashboard for managing all 29 configurable YAML parameters of the Netty-Proxy system. Provides real-time status monitoring, dynamic config editing with live reload, and visual route rule management. Replaces manual YAML editing for day-to-day operations.

## Brand Personality

Precise. Dense. Functional. A tool that disappears into the task. Reads like a well-tuned terminal: every pixel earns its place, no decoration without information.

## Anti-references

- SaaS dashboards with gradient hero sections and card grids
- Purple-to-blue gradients, glassmorphism panels
- Rounded-square icon tiles above headings
- Marketing-style dashboards that prioritize looks over density

## Design Principles

1. Information density over whitespace. Show everything, let the user decide what to focus on.
2. Consistent component vocabulary. Same form controls, same spacing, same feedback patterns.
3. State visibility. Every connection, every config change, every parameter has a clear state indicator.
4. Direct manipulation. Edit in place, apply immediately, see results.

## Accessibility & Inclusion

WCAG AA minimum. High contrast dark theme. Keyboard navigable forms. Reduced motion respected.
